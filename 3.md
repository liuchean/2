# 鳥哥第二堂課

## 1.指令基本概念
### 1.1 指令基本格式
在 Linux 的文字界面（Bash Shell）中，下達指令時必須遵循特定的語法結構。標準的指令下達方式如下：

`[student@localhost ~]$ command  [-options..]  [parameter1...]`
* **語法提示**：括號 `[ ]` 表示該項目是**選填**的（可有可無）。
#### 結構說明：
* **command (指令)**：
    * 一行指令的第一個部分。
    * 可以是指令名稱（如 `history`）、可執行檔案或腳本（script）。
* **[-options] (選項)**：
    * **短選項**：通常帶有一個減號 `-`，例如 `-h`。
    * **長選項**：通常帶有兩個減號 `--`，例如 `--help`。
    * **注意**：`-help` 通常會被系統拆解為 `-h -e -l -p` 四個單獨選項，與 `--help` 不同。
* **[parameter] (參數)**：
    * 依附在選項後或指令後的資料。
    * 若表示為 `parameters...`（帶有小數點），代表可以輸入多個參數。

---

### 1.2 指令執行注意事項
為了確保指令正確執行，請務必留意以下規則：

1.  **空格區隔**：指令、選項與參數之間必須以**空格**或 `[Tab]` 鍵隔開。不論空幾格，系統皆視為一格。
2.  **啟動執行**：按下 `[Enter]` 鍵後，系統才會開始辨識並執行該行指令。
3.  **大小寫敏感**：Linux 嚴格區分英文大小寫。例如 `cd` (切換目錄) 與 `CD` 會被視為完全不同的指令。
4.  **特殊組合**：有時選項會帶有參數，常見語法如 `[-option para]` 或 `--option=para`。
   
---

### 1.3 輔助文件說明 (以 date 為例)

當我們使用 `--help` 或 `man` 查詢指令用法時，輔助文件通常包含以下核心區塊：

#### 1.3.1 Usage (執行語法)
定義了指令的各種使用情境，常見格式如下：
* `date [OPTION]... [+FORMAT]`：用於**顯示**目前時間，並可自定義輸出格式。
* `date [-u|--utc] [MMDDhhmm[[CC]YY][.ss]]`：用於**設定**系統時間。

#### 1.3.2 Mandatory arguments (主要選項說明)
列出指令可使用的參數及其用法。
* **範例**：`-d, --date=STRING`
    * 可使用短選項 `-d STRING` 或長選項 `--date=STRING`。
    * `STRING` 代表你指定的日期。
    * **實作**：若要顯示昨天日期，可下達 `date -d yesterday`。

#### 1.3.3 FORMAT controls (格式化控制)
當使用 `+FORMAT` 語法時，可以自定義顯示的內容，常見代碼包括：
* `%a`：星期的英文縮寫（小寫）。
* `%A`：星期的英文全稱（大寫）。
* `%Y`：完整西元年（如 2024）。
* `%m`：月份（01..12）。
* `%d`：日期（01..31）。
* **例子**：輸入 `date +%Y-%m-%d` 可輸出如 `2026-05-26`。

---

### 1.4 身份切換 su - 的使用

#### 1.4.1 權限限制實作：以設定日期為例
在 Linux 中，一般帳號無法修改系統級參數。若嘗試設定時間：
* **指令**：`date MMDDhhmmYYYY`
* **範例**：`date 021916462023`
* **結果**：出現 `date: 無法設定日期: 此項操作並不被允許` (Operation not permitted)。

#### 1.4.2 使用 su - 切換至管理員 (root)
若要執行管理任務，需切換身份：
1. **輸入指令**：`su -`
2. **輸入密碼**：輸入密碼 (輸入時螢幕不會顯示任何字元)。
3. **切換成功**：提示字元由 `$` 變為 `#` 即代表成功切換。

切換 root 後，即可完成操作，但在做完任何系統維護行為後，請回復到一般用戶的身份較佳 (這個習慣請務必養成！)。

---

### 1.5 常見的熱鍵與組合按鍵
在純文字模式 (Bash Shell) 的環境下，熟練使用熱鍵可以大幅提升操作效率：

* **`[Tab]` (最重要)**：
    * **命令補齊**：輸入指令前幾個字母後按一下，可自動補齊指令。
    * **檔案補齊**：在指令後輸入路徑或檔名時，可自動補齊檔名。
    * **變數補齊**：亦可用於補齊環境變數名稱。
* **`[Ctrl]` + `c`**：
    * **中斷指令**：當某個指令執行太久或發生錯誤時，可用來強行中斷目前運作中的任務。
* **`[Shift]` + `[PageUp]` / `[PageDown]`**：
    * **翻頁功能**：在純文字界面中，用來向上或向下捲動螢幕畫面，查閱先前的輸出資訊。
---
### 1.6 man (manual page)指令操作
在 Linux 裡，man 指令就是 manual page 的縮寫，用來查詢系統提供的「說明手冊」。它比 --help 更完整，幾乎是所有指令、函式庫、設定檔的百科全書。

進入 `man` 指令的說明畫面後，可以使用下列按鍵來移動與搜尋內容：
#### 1.6.1 畫面移動
* **`[Enter]`**：向後移動一行。
* **方向鍵 `[上]` / `[下]`**：向前 / 向後移動一行。
* **`[PageUp]` / `[PageDown]`**：向前 / 向後移動一頁。
* **`g`**：移動到整份文件的**第一行**。
* **`G`**：移動到整份文件的**最後一行**。
* **`q`**：**離開** man page 畫面。

#### 1.6.2. 關鍵字搜尋
若要快速查找特定項目，可使用搜尋功能：
* **`/keyword`**：輸入斜線 `/` 後接關鍵字，即可向下搜尋。
* **`n`**：繼續找**下一個**符合的關鍵字。
* **`N`**：回頭找**上一個**符合的關鍵字。
#### 1.6.3 man page 的章節編號意義
在 man page 的第一行通常會看到指令名稱後跟著一個括號數字，例如 `BC(1)`。這個數字代表該文件所屬的分類：

| 代號 | 代表內容 |
| :--- | :--- |
| **1** | 使用者在 Shell 環境中可以操作的指令或可執行檔 |
| **2** | 系統核心（Kernel）可呼叫的函數與工具等 |
| **3** | 常用的函數（Function）與函式庫（Library），多為 C 函式庫 |
| **4** | 裝置檔案的說明（通常位於 `/dev` 下的檔案） |
| **5** | 設定檔或某些檔案的格式（例如 `/etc/passwd`） |
| **6** | 遊戲 (Games) |
| **7** | 慣例與協定（如 Linux 檔案系統、網路協定、ASCII code 等） |
| **8** | 系統管理員可用的管理指令（通常需要 root 權限） |
| **9** | 與 Kernel 原始碼相關的文件 |


**例子**：可以輸入 `man passwd` 跟輸入 `man 5 passwd`，就可以查詢兩個相同檔名，但不同位置與功能的說明文件了！

## 2. 檔案管理
### 2.1 檔案系統階層標準 (FHS) 與目錄結構

Linux 的檔案配置遵循 **FHS (Filesystem Hierarchy Standard)** 標準，確保軟體與使用者能預期檔案存放的位置。

#### 2.1.1 檔案內容分類
根據檔案的性質，可分為以下四類：
* **Static (固定)**：不隨系統運作變動的工具程式、函式庫與說明文件。
* **Dynamic (變動)**：運作中會被程序修改的資料。
* **Persistent (持續)**：重啟後依然存在的資料（如設定檔）。
* **Runtime (運作)**：程序運行中的資料，重啟後會被刪除。

#### 2.1.2 核心目錄說明 (必知項目)
可使用指令 `ll /` 觀察系統實際目錄：

| 目錄名稱 | 應放置的檔案內容 |
| :--- | :--- |
| **`/bin`**, **`/sbin`** | **指令區**：`/bin` 放一般指令，`/sbin` 放管理指令。在 RHEL 9 中均連結至 `/usr/bin` 與 `/usr/sbin`。 |
| **`/boot`** | **開機檔**：包含核心 (Kernel) 與開機管理程式 (Grub) 相關檔案。 |
| **`/dev`** | **裝置檔**：系統所有硬體裝置（如硬碟、終端機）均以此處的檔案代表。 |
| **`/etc`** | **設定檔**：帳號、密碼及各種服務的設定檔。 |
| **`/home`**, **`/root`** | **家目錄**：`/home` 為一般用戶，`/root` 為管理員專用。 |
| **`/lib`**, **`/lib64`** | **函式庫**：系統與核心所需的函式庫，現多連結至 `/usr/lib` 與 `/usr/lib64`。 |
| **`/proc`**, **`/sys`** | **虛擬檔案系統**：將記憶體與硬體資訊映射成檔案，可即時查閱或調整參數。 |
| **`/usr`** | **軟體資源**：Unix Software Resource，系統主要的軟體、服務均存放於此。 |
| **`/var`** | **變動資料**：包含系統日誌 (Log)、郵件、資料庫等經常變動的檔案。 |
| **`/tmp`** | **暫存區**：放置執行中的暫存檔案。 |

#### 2.1.3 其他目錄 (進階了解)
* **`/media` / `/mnt`**：`/media` 用於隨插即用裝置(USB)等；`/mnt` 用於手動掛載。
* **`/opt`**：第三方協力廠商軟體安裝處。
* **`/run`**：系統服務運作管理功能（存放於記憶體中）。
* **`/srv`**：放置各類服務 (Service) 所需的資料。
---
###2.2 絕對路徑相對路徑
* **絕對路徑**：在 Linux 中，所有目錄皆由「根目錄 `/`」衍生，從 `/` 開始撰寫的完整路徑即為絕對路徑。
#### 2.2.1 磁碟觀察(df)
* **觀察磁碟空間**：
    * **指令**：`df` (Display Filesystem)
    * **功能**：查閱磁碟分割區與目錄樹的對應關係。

 #### 2.2.2 工作目錄與路徑切換(cd, pwd)
 * **`pwd` (Print Working Directory)**：顯示目前所在的**絕對路徑**。
    > [!IMPORTANT]
    > 提示字元可能無法顯示完整路徑，操作前要使用 `pwd` 確認目前所在位置，避免改錯檔案。
* **`cd` (Change Directory)**：切換工作目錄。
    * 範例：`cd /var/spool/mail` 可將工作目錄移動至該處。
#### 2.2.3 絕對路徑 vs 相對路徑

在路徑寫法中，以下這些符號代表特定的位置：
| 符號 | 意義 |
| :--- | :--- |
| **`/`** | **根目錄**：系統的最頂層。 |
| **`~`** | **家目錄**：目前登入使用者的個人目錄。 |
| **`.`** | **本目錄**：代表目前所在的工作目錄。 |
| **`..`** | **上一層目錄**：代表目前目錄的父目錄。 |
| **`-`** | **前一個工作目錄**：切換回上一次停留的位置。 |

根據起點的不同，路徑寫法分為兩類：
* **絕對路徑 (Absolute Path)**：
    * 必須由根目錄 **`/`** 開始寫起。
    * 例如：`/home/student/.bashrc`
* **相對路徑 (Relative Path)**：
    * 相對於「目前路徑」的寫法。
    * **開頭不是 `/`**，通常以 `.` 或 `..` 開頭。
    * 例如：`./text1.txt` 或 `../../etc`
