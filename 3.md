## 1.檔案管理
在 Linux 底下，所有的東西都以檔案來呈現，不同的檔案特性會有不同的結果。讀者可以常見的兩種檔案格式為

### 1.1 目錄的建立與刪除
在 Linux 中，管理目錄最基礎的兩個指令為 `mkdir` 與 `rmdir`：

* **`mkdir` (Make Directory)**：
    * **功能**：建立一個新的「空目錄」。
    * **特性**：新建立的目錄內不會有任何檔案。
    * **範例**：`mkdir mydata`

* **`rmdir` (Remove Directory)**：
    * **功能**：刪除一個「空目錄」。
    * **限制**：若目錄內含有其他檔案或子目錄，`rmdir` 將無法執行刪除。
    * **範例**：`rmdir mydata`

> [!NOTE]
> 實務上若要刪除「非空目錄」（包含內容物一併刪除），通常會使用 `rm -r` 指令，但預設 rm 僅能刪除一般檔案無法刪除目錄。

### 1.2 萬用字元
在 Bash 環境中，當我們想要查詢特定關鍵字的檔案時，可以利用「萬用字元」來快速篩選。
| 符號 | 意義 |
| :--- | :--- |
| **`*`** | 代表 **0 個到無窮多個**任意字元。 |
| **`?`** | 代表 **一定有一個**任意字元。 |
| **`[ ]`** | 代表 **一定有一個**在括號內的字元。例如 `[abcd]` 指的是 a, b, c, d 其中之一。 |
| **`[ - ]`** | 代表在編碼順序內的**所有連續字元**。例如 `[0-9]` 代表所有數字。 |
| **`[^ ]`** | 代表**反向選擇**。例如 `[^abc]` 代表「只要不是 a, b, c」的其他字元都接受。 |
ex ：查詢 /etc 底下開頭為 cron 的檔案
若要尋找檔名開頭為 `cron` 的資料，可以使用：ll -d /etc/cron*

[!TIP]
若不加 -d：當匹配到「目錄」時，ll 會列出該目錄下的所有子檔案，導致畫面混亂。
若加上 -d：僅會列出目錄本身的屬性，讓輸出結果更直觀。

### 1.3 檔案及目錄的複製(cp)刪除(rm)
#### 1.3.1 檔案與目錄的複製 (cp)

`cp` 指令預設僅複製檔案，若要處理目錄或保留完整屬性，需使用特定選項：

* **常用選項**：
    * `-r`：用於**複製目錄**，複製後的檔案時間會變成「目前時間」。。
    * `-a`：相當於 `-pdr` 的集合，能**完整保留**權限、連結檔屬性及檔案時間（常用於備份）。

* **連結檔 (Link) 的複製特性**：
    * 連結檔在 `ll` 輸出中開頭為 `l`，並指向原始檔案（如 `mail -> spool/mail`）。
    * **技巧**：若要複製連結檔指向的「實際內容」而非連結檔本身，目錄結尾必須加上斜線 `/`。
        * 範例：`cp -r /var/mail/ .`


#### 1.3.2 複製目標的存在與影響
根據 `cp` 的語法，目的地的狀態會影響結果：
1. **目標目錄不存在**：系統會建立一個與來源內容相同的新目錄。
2. **目標目錄已存在**：系統會將整個來源目錄**搬移到**目標目錄之內。
    * *範例*：重複執行 `cp -r /etc/rc.d rc.d`，第二次執行後，路徑會變成 `rc.d/rc.d/`。


#### 1.3.3 刪除檔案與目錄 (rm)

刪除動作在 Linux 中通常是**不可逆**的，操作時需小心。

* **常用指令與選項**：
    * `rm [file]`：刪除一般檔案。
    * `-r`：遞迴刪除（用於目錄）。
    * `-f`：強制刪除，忽略不存在的檔案且不詢問。

### 1.4 特殊檔名

#### 1.4.1 檔名中間有空白字元
一般可以使用單引號或雙引號或反斜線 (\) 來處理這樣的檔名。
ex: 如果要創建一個『 class one 』的檔名時 => mkdir "class one"

如果要刪除使用rmdir class one 會判定為有兩個名為 class 與 one 的目錄要刪除，因為找不到，所以回報錯誤，此時使用雙引號或單引號來處理， 也可以透過反斜線將空白變成一般字元即可，如rmdir class\ one

#### 1.4.2 加號與減號開頭的檔名
直接給mkdir -newdir 會報錯，嘗試使用單引號來處理時，同樣回報錯誤！使用反斜線，同樣回報錯誤。

如果真的要建立此檔案，要使用絕對 or 相對路徑來解決
mkdir /dev/shm/-newdir or mkdir ./-newdir2

### 1.5 檔名移動與改名(mv)
若檔案建立到錯誤的位置時，可以使用 mv 來處理。同時若檔名鍵錯，也能夠使用 mv 來更名。

## 2. 檔案內容查詢
很常使用者只是需要知道檔案內容，並沒有進行編輯。此時可以透過一些簡易的指令來查詢文件檔案的內容。

### 2.1 基礎查閱指令 (cat, head, tail)

* **`cat` (Concatenate)**：
    * **功能**：將檔案內容**全部**列出在螢幕上。
    * **缺點**：若檔案過大，螢幕會快速捲動，難以閱讀。
* **`head`**：
    * **功能**：預設僅列出檔案的**最前面 10 行**。
    * **範例**：`head -n 5 filename` (只看前 5 行)。
* **`tail`**：
    * **功能**：預設僅列出檔案的**最後面 10 行**。
    * **應用**：非常適合用來查看系統日誌（Log）的最新資訊。
    * **範例**：`tail -n 5 filename` (只看最後 5 行)。

---

### 2.2 可檢索檔案內容
當檔案資料量較大時，使用前述方法會導致畫面快速捲動。此時建議使用 `more` 或功能更強大的 `less` 來進行分頁閱讀。

#### 2.2.1 more：基礎分頁
* **特性**：預設僅能一頁一頁「向後」翻動。
* **常用指令**：
    * `空白鍵`：向下/向後翻一頁。
    * `/關鍵字`：在文件中搜尋關鍵字。
    * `q`：離開查閱畫面。

#### 2.2.2 less：進階分頁
* **特性**：支援向前與向後翻頁。事實上，`man page` 預設就是呼叫 `less` 進行處理。
* **常用指令**：
    * `空白鍵` 或 `[PageDown]`：向下/向後翻頁。
    * `[PageUp]`：向上/向前翻頁。
    * `g`：直接跳到檔案的**第一行**。
    * `G`：直接跳到檔案的**最後一行**。
    * `/關鍵字`：向下搜尋關鍵字。
    * `q`：離開查閱畫面。

## 3. Vim 編輯器

使用者經常需要變動系統設定檔或進行純文字編輯，此時 `vi` 或 `vim` 的支援就非常重要。

* ** why Vim？**
    * **通用性**：它是許多 Linux 指令預設呼叫的編輯程式。
    * **系統管理**：在沒有圖形介面的環境下，它是唯一的修補工具。
* **vi vs. vim**：
    * **vi**：基礎的文書編輯器。
    * **vim**：vi 的進階版，增加了**顏色顯示 (Syntax Highlighting)** 與更強大的程式編輯支援。


### 3.1 vim 基本模式

Vim 的操作邏輯與一般編輯器不同，它是透過「模式切換」來執行不同的任務。以下是四種最基本的模式：
#### 1. 一般指令模式 (Command Mode)
* **進入方式**：輸入 `vim filename` 進入後的初始狀態。
* **功能**：移動游標、複製 (`y`)、刪除 (`d`)、貼上 (`p`)、復原 (`u`) 等。
* **注意**：此模式下無法直接打字。

#### 2. 編輯模式 (Insert/Edit Mode)
* **進入方式**：在一般模式按下 **`i`** (insert)。
* **功能**：開始進行文字內容的輸入與編輯。
* **回到一般模式**：按下 **`Esc`** 鍵。

#### 3. 延伸指令列命令模式 (Extended Command Mode)
* **進入方式**：在一般模式按下冒號 **`:`**。
* **功能**：
    * 儲存與離開：`:w` (存檔)、`:q` (離開)、`:wq` (存檔並離開)。
    * 強制動作：加上 `!` (如 `:q!` 為不存檔強制離開)。
    * 其他：搜尋與取代。

#### 4. 選取模式 (Visual Mode)
* **功能**：類似滑鼠圈選功能，選取後可按 `y` 複製或 `d` 刪除。
* **進入方式**：
    * **`v`**：以「字元」為單位進行圈選。
    * **`Shift` + `v`**：以「整行」為單位進行圈選。
    * **`Ctrl` + `v`**：以「區塊 (矩形)」為單位進行圈選。

> [!TIP]
> 如果迷失在任何模式中，連按兩次 **`Esc`** 鍵通常可以讓你回到「一般指令模式」。
