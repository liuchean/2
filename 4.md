## 1. Linux權限
### 1.1 使用者、群組與其他人
Linux 的檔案權限主要依據三種身份來決定，這對系統安全與檔案管理至關重要：

#### 1. 三種身份定義
* **Owner (使用者/擁有者)**：檔案的所屬人，擁有最高控制權。
* **Group (群組)**：檔案附屬的團隊，加入該群組的帳號皆享有群組權限。
* **Others (其他人)**：既不是擁有者，也沒有加入對應群組的帳號。

#### 3. 管理核心邏輯
Linux 的權限管理是**針對帳號**進行的。為了簡化管理，系統將非本人的所有帳號分為兩類：
1. **群組成員**：方便進行團隊協作或權限控管（如班級、部門）。
2. **其他人**：不屬於特定團隊的外部帳號。

> [!NOTE]
> 這種權限結構，讓管理員可以精準控制誰能讀取、修改或執行特定檔案 。

### 1.2 觀察檔案權限
使用 ls -ld /var/spool/mail 可獲得以下資訊
drwxrwxr-x. 2 root mail 21    Feb 16 11:51 /var/spool/mail

| **A** | `drwxrwxr-x.`(檔案類型與權限，包含 1 個類型字元 + 9 個權限字元 (rwx))

| **B** | `root` => 表示擁有者

| **C** | `mail` => 表示群組

| **D** | `Feb 16 11:51` => 表示修改日期(檔案最後變更時間)

| **E** | `/var/spool/mail`=>檔名或目錄名稱

> [!TIP]
> 緊接在類型之後的 9 個字元，每 3 個一組，分別代表「擁有者、群組、其他人」的權限：
* **`r` (read)**：可讀取檔案內容或列出目錄清單。
* **`w` (write)**：可編輯檔案內容或在目錄內新增/刪除檔案。
* **`x` (execute)**：可執行該檔案或**進入該目錄**。
* **`-`**：表示無該項權限。
> 記得這 9 個字元的順序永遠是：**`[擁有者(rwx)][群組(rwx)][其他人(rwx)]`**。

如果更清楚的查看檔案的相關屬性與權限可使用 getfacl 指令，但就不會有檔案類型，修改時間等參數。

```bash
[student@localhost ~]$ getfacl /var/spool/mail
getfacl: Removing leading '/' from absolute path names
# file: var/spool/mail   <== 檔名
# owner: root            <== 檔案擁有者
# group: mail            <== 檔案群組
user::rwx                <== 使用者的權限
group::rwx               <== 同群組帳號的權限
other::r-x               <== 其他人的權限
```

### 1.2 修改檔案擁有者 群組 權限
#### 1.2.1修改檔案擁有者(chown)

在進行權限修改前，先確認系統中是否有名為 `daemon` 的帳號：
```bash
[root@localhost shm]# id daemon
uid=2(daemon) gid=2(daemon) groups=2(daemon)
```
若帳號存在，使用 chown 指令將 checking 的擁有者由 student 改為 daemon：
```bash
[root@localhost shm]# chown daemon checking
[root@localhost shm]# ll checking
-rw-r--r--. 1 daemon student 0 Feb 21 22:54 checking
```

#### 1.2.2修改檔案擁有的群組(chgrp)
系統的群組都紀錄在 /etc/group 檔案內，若想要了解系統是否存在某個群組，可以使用 grep 這個關鍵字擷取指令來查詢。

```bash
[root@localhost shm]# grep myname /etc/group
# 不會出現任何資訊，因為沒有這個群組存在的意思。

[root@localhost shm]# grep bin /etc/group
bin:x:1:        <==代表確實有這個群組存在！
```
修改群組
```bash
[root@localhost shm]# chgrp bin checking
[root@localhost shm]# ll checking
-rw-r--r--. 1 daemon bin 0 Feb 21 22:54 checking
```
#### 1.2.3 搭配數字法修改權限 (chmod)
由於檔案紀錄了三種身份，每種身份都擁有 `rwx` 的最大權限與 `---` 沒權限的情況。為了記憶與計算方便，系統採用 **2 進位** 的邏輯來轉換為數字：

* **`r` (read)** ：$2^2 = 4$
* **`w` (write)** ：$2^1 = 2$
* **`x` (execute)** ：$2^0 = 1$

每一種身份的權限分數為這三個數值的加總：
* **最低分**：`0` (---)
* **最高分**：`7` (r+w+x，即 4+2+1)

因為共有三種身份（使用者、群組、其他人），權限組合最高為 `777`，最低為 `000`。

**範例說明：**
以上述 `checking` 檔案權限 `-rw-rw-r--` 為例：
1. **使用者 (User)**：`rw-` = 4 + 2 + 0 = **6**
2. **群組 (Group)**：`rw-` = 4 + 2 + 0 = **6**
3. **其他人 (Others)**：`r--` = 4 + 0 + 0 = **4**
> 該檔案的權限數字表示即為 **`664`**。
```bash
[root@localhost shm]# chmod 664 checking
[root@localhost shm]# ll checking
-rw-rw-r--. 1 daemon bin 0 Feb 21 22:54 checking
```

## 2.群組共用
某些情境下，群組可能需要共享某些檔案資料。舉例來說， progroup 成員 prouser1, prouser2, prouser3 ，需要共用 /srv/project1/ 的目錄， 則該目錄的建置與共享可以使用如下的方式來達成：
```bash
# 建立專案目錄
[root@localhost ~]# mkdir /srv/project1

# 修改目錄所屬群組為 progroup
[root@localhost ~]# chgrp progroup /srv/project1

# 設定權限：擁有者與群組皆為 rwx (7)，其他人無權限 (0)
[root@localhost ~]# chmod 770 /srv/project1

# 確認目錄狀態
[root@localhost ~]# ls -ld /srv/project1
drwxrwx---. 2 root progroup 6 Feb 22 10:26 /srv/project1/
```
