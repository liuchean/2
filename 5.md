# 鳥哥第5堂課

## 1.權限在目錄與檔案應用上的意義

### 1.1 權限對檔案的重要性
檔案是實際含有資料的地方，包括一般文字檔、資料庫內容檔、二進位可執行檔 (binary program) 等。權限對於檔案的意義如下：

* **r (Read)**：可讀取此一檔案的實際內容。
* **w (Write)**：可以編輯、新增或者是修改該檔案的內容 (**但不含刪除該檔案**)。
* **x (eXecute)**：該檔案具有可以被系統執行的權限。

> [!IMPORTANT]
> 對於檔案的 rwx 來說，主要都是針對**「檔案的內容」**，與檔案檔名的存在與否沒有關係。並且 `w` 權限並不具備刪除檔案本身的權限。

---

### 1.2 權限對目錄的重要性
目錄主要的內容在於**記錄檔名清單**。針對目錄rwx 的意義如下：

* **r (read contents in directory)**：
    具有讀取目錄結構清單的權限，代表用戶可以查詢該目錄下的檔名資料（例如執行 `ls`）。
* **w (modify contents of directory)**：
    具有異動該目錄結構清單的權限，包括：
    * 建立新的檔案與目錄。
    * **刪除已經存在的檔案與目錄**
    * 將已存在的檔案或目錄進行更名。
    * 搬移該目錄內的檔案、目錄位置。
* **x (access directory)**：
    目錄的 `x` 代表的是使用者能否**進入該目錄**成為工作目錄（例如執行 `cd`）。

> [!IMPORTANT]
> 如果要對目錄下的檔案進行刪除，關鍵在於該「目錄」是否具備 `w` 權限，而不是該檔案本身的權限。

## 2. 基礎程序管理
在 Linux 系統運作中，所有在系統上面運作的都是透過觸發程式成為記憶體中的程序後，才能夠順利的操作系統。

###2.1 程式和程序關係

程式 (Program) + 執行行為 (Execution) = 程序 (Process)

---

### 2.2 父程序和子程序
程序之間具有相依性。當一個程序觸發另一個程序時，觸發者為**父程序 (Parent)**，被觸發者則為**子程序 (Child)**。

#### 身份切換舉例 (su 指令)
當我們使用 `su -` 切換身份時：
1.  原本登入系統的 **bash** 是父程序。
2.  執行 `su -` 指令後，系統會派生出一個具備 root 權限的新 **bash**，這個新 bash 就是子程序。
   
> [!IMPORTANT]
> 這種層級關係在 Linux 系統管理中非常重要，如果父程序被終止，通常其下的子程序也會隨之結束或受到影響。

---

### 2.3 程序的優先序 PRI 與 NI 

因記憶體內的有大量的程序，所以在系統執行程序的時候會被加上優先級Priority (PRI)，其值越低代表系統會執行越多次此程序，也就代表該程序會越早被執行完(同一周期內執行多次)。

雖然 `PRI` 是由系統自行規劃的，使用者無法直接修改，但可以透過修改 **Nice (NI)** 值來「影響」`PRI`。其基本關係式如下：

$$PRI_{(new)} = PRI_{(old)} + NI$$

* **NI 值越小**：`PRI` 隨之變小，程序的優先序會**往前挪**（優先執行）。
* **NI 值越大**：`PRI` 隨之變大，程序的優先序會**變落後**。

#### 2.3.1 Nice 值的使用限制
為了避免一般用戶搶佔系統資源，NI 的調整有以下限制：
* **範圍**：調整範圍為 `-20` ~ `19`。
* **root 權限**：可隨意調整(renice)自己或他人的 Nice 值，範圍為 `-20` ~ `19`。
* **一般使用者**：
    * 僅可調整自己程序的 Nice 值。
    * 調整範圍僅限 `0` ~ `19`。
    * **只能調高不能調低**（例如原本 NI 為 5，之後只能調到大於 5）。

#### 2.3.2 如何調整 NI 值
主要透過以下三種工具進行：
1. **`nice` 指令**：在一開始執行程式時，就給予一個特定的 nice 值。
2. **`renice` 指令**：調整已經存在（運作中）之 PID 的 nice 值。
3. **`top` 指令**：在互動式界面中，直接修改運作中程序的 nice 值。

### 2.4 bash 的工作管理
**「當用戶登入系統取得 bash shell 之後，在單一終端機介面下同時進行多個工作的行為管理」**。
例如：您可以同時在背景複製檔案、搜尋資料，而前景持續使用 vim 撰寫程式。

#### 2.4.1 工作管理的限制與特性
進行 Job Control 時，必須注意以下限制：
* **範圍限制**：這些工作所觸發的程序必須來自於你當前 shell 的**子程序**（即「只管理自己的 bash」）。
* **前景 (Foreground)**：目前正在控制、可直接下達指令的環境。
* **背景 (Background)**：
    * 可以自行運作，無法使用 `[Ctrl]+c` 終止。
    * 需使用 `fg` 或 `bg` 指令呼叫。
    * **背景執行的程序不能等待 Terminal 的輸入 (input)**。

#### 2.4.2 常用的管理指令與快捷鍵
| 指令/按鍵 | 功能說明 |
| :--- | :--- |
| **`command &`** | 直接將指令丟到背景執行。 |
| **`[Ctrl]+z`** | 將目前正在前景的工作丟到背景中**暫停**。 |
| **`jobs [-l]`** | 列出目前背景中所有的工作資訊。(`-l` 可列出 PID) |
| **`fg %n`** | 將編號為 `n` 的背景工作移回**前景**操作。 |
| **`bg %n`** | 將編號為 `n` 的背景工作在背景中恢復**執行**。 |

### 2.5 特殊權限 SUID/SGID/SBIT 的功能

#### 2.5.1 SUID的基本功能與限制

假設系統的密碼紀錄在 `/etc/shadow` 內，一般使用者並沒有權限讀取或更改。但為了讓用戶能修改自己的密碼，系統使用了 **Set UID (SUID)** 技術。

當系統在 `passwd` 執行檔上設定了 SUID 權限旗標，使用者執行該指令時，就能暫時獲得該程式擁有者（通常是 root）的權限。

* **對象限制**：SUID 權限僅對**二進位程式 (binary program)** 有效，不能用在 shell script。
* **執行權限**：執行者對於該程式必須先具有 `x` (可執行) 的權限。
* **時效性**：本權限僅在**執行該程式的過程中**有效 (run-time)。
* **權限提升**：執行者在執行期間將具有該程式**擁有者 (owner)** 的權限。

**舉例：passwd 指令**
當一般用戶執行 `/usr/bin/passwd` 時，因為該檔案具有 SUID，該用戶會暫時取得 root 的權限，進而能夠寫入原本無權存取的 `/etc/shadow` 檔案以完成密碼修改。


#### 2.5.2 SGID 的功能

與 SUID 類似，**SGID (Set Group ID)** 是將特殊的權限旗標設定在群組（Group）的 `x` 權限上。當 SGID 設定在**檔案**時，其功能如下：

* **對象限制**：SGID 對**二進位程式 (binary program)** 有效。
* **執行權限**：程式執行者對於該程式來說，需具備 `x` 的可執行權限。
* **權限提升**：執行者在執行的過程中，將會獲得該程式**「群組」**的支援（亦即執行者將暫時擁有該程式所屬群組的權限）。
=> 簡單來說，SUID 是讓執行者變成「檔案擁有者」，而 SGID 則是讓執行者變成「檔案群組成員」。

#### 2.5.3 SBIT 的功能

`/tmp` 是所有帳號均可寫入的一個暫存目錄，因此理論上應該是 `777` 的權限。但若是純粹的 `777`，代表任何人建立的資料都可能被他人隨意刪除。

為了避免這種情況，`/tmp` 會加上 **Sticky Bit (SBIT)** 的特殊權限旗標。

#### SBIT 的功能與限制
* **適用對象**：SBIT 目前僅針對**目錄**有效。
* **觸發條件**：使用者對於此目錄必須具有 `w` 與 `x` 權限（即具有寫入權限）。
* **保護機制**：當使用者在該目錄下建立檔案或目錄時，**僅有「檔案擁有者」與「root」才有權力刪除該檔案**。

=> 簡單來說，SBIT 的作用是：「雖然大家都能在裡面建立檔案，但除了 root 之外，你只能刪除你自己建立的東西，不能刪別人的。」

#### 2.5.4 SUID/SGID/SBIT 權限的設定
SUID/SGID/SBIT 的權限旗標是在 Linux 傳統「三個身份、三個權限」之外的隱藏屬性，因此在數字表示法中，會產生 **第四個位數**（放在最左側）。

#### 1. 特殊權限的分數計算
這三個特殊權限的分數計算方式與一般權限類似，也是採用 4-2-1 的累積制：

* **4**：代表 **SUID**
* **2**：代表 **SGID**
* **1**：代表 **SBIT**

#### 2. 設定範例
若要設定權限，只需將特殊權限的分數加在原本的三位數權限之前。例如：

* **`chmod 4755 filename`**：
  設定為 SUID (4)，一般權限為 `rwxr-xr-x` (755)。
* **`chmod 2770 dirname`**：
  設定為 SGID (2)，一般權限為 `rwxrwx---` (770)。
* **`chmod 1777 /tmp`**：
  設定為 SBIT (1)，一般權限為 `rwxrwxrwx` (777)。
